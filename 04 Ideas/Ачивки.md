## 📋 Техническое задание: RIS-философия и переименование ачивок

### 1. Цель

Сдвинуть тональность Syverro с **игровой / ироничной** на **аналитическую / рефлексивную** в соответствии с концепцией **Reading Intelligence System (RIS)**.

### 2. Что нужно сделать

#### 2.1. Переименовать ачивки

| Было (ироничное) | Стало (RIS) |
|------------------|--------------|
| «Ну, началось» | «Первый шаг» |
| «Читатель» | «Первая завершённая книга» |
| «Золотая звезда» | «Максимальная оценка» |
| «Критик с дивана» | «Первая рецензия» |
| «Уже не остановить» | «Десять книг» |
| «Пора лечиться» | «Пятьдесят книг» |
| «Библиотекарь-самоучка» | «Сто книг» |
| «Разогрев» | «Пятьсот страниц» |
| «Многостраничник» | «Тысяча страниц» |
| «Полиглот-недоучка» | «Три языка» |
| «Жанровый хаос» | «Пять жанров» |

| «Олдскул» | «Историческая перспектива» |
| «Зумер» | «Современный контекст» |

#### 2.2. Изменить интерфейс

- Убрать лишние эмодзи / смайлики в серьёзных местах (статистика, профиль)
- Добавить **метрики** вместо «наград»:
  - Индекс жанрового разнообразия
  - Литературный хронотип (по времени чтения)
  - Календарь чтения (тепловая карта)
- Сделать экран ачивок более «дашбордным», менее игровым

#### 2.3. Обновить тексты в интерфейсе

- Заменить «Достижения» на «Метрики» или «Профиль читателя» (позже решить)
- В статистике добавить пояснения: *«Что означают эти цифры?»*

#### 2.4. Подготовить базу для новых RIS-фич

- Добавить в книгу поле **«Время чтения (минуты)»** (пока опционально)
- Добавить в книгу поле **«Теги / настроение»** (пока не обязательно)

### 3. Где менять

| Файл | Что менять |
|------|------------|
| `locales/*.js` | Названия ачивок, возможно, раздел `achievements` |
| `store/achievements.js` | Массив `ACHIEVEMENTS_LIST` (только названия) |
| `screens/AchievementsScreen.js` | UI (убрать лишние эмодзи, добавить метрики) |
| `screens/ProfileScreen/index.js` | Добавить RIS-метрики (индекс разнообразия) |
| `screens/StatsScreen/index.js` | Добавить «литературный хронотип» (если есть данные) |

### 4. Ограничения

- **Не трогать логику проверки ачивок** — только названия и отображение
- **Не удалять старые ID ачивок** — иначе сломается сохранённый прогресс
- **Не менять архитектуру** — только косметика и новые метрики поверх

### 5. Приоритет внутри задачи

1. **Переименовать ачивки в коде и локалях** (безопасно, меняются только строки)
2. **Убрать лишние смайлики** на экране ачивок и статистики
3. **Добавить индекс жанрового разнообразия** в профиль (новая метрика)
